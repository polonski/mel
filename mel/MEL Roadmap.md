# MEL Roadmap

The ideal evolution for [[Machine Expression Language (MEL)]] transitions it from a static prompt generation tool into a dynamic, self-improving orchestration language. Based on current vision documents, here is the synthesized roadmap:

## 1. Deep IDE Integration & Scaffolding
Before scaling out, MEL must scale *in* to the developer workflow. 
- **Intelligent Scaffolding:** Contextually analyzing the developer's initial query or code state to pre-set directives (e.g., defaulting to "Literary" style for poetry, or "Senior Engineer" for code).
- **Direct AI Interface:** Routing high-precision prompts natively into IDE tools like [[Gemini Code Assist]].

## 2. Extensibility & Exporting
- **Dynamic Directives:** Allowing users to define custom sliders (e.g., an `Audience` slider ranging from "a 5-year-old" to "a panel of PhDs").
- **Pipeline Compilation:** Expanding the "Export As..." functionality to natively compile MEL into production-ready formats like DSPy Signatures, LangChain Prompt Templates, and Claude's XML tool-use formats.

## 3. Community Ecosystem
- **MEL Packs:** Enabling the community to curate, save, and share domain-specific vocabularies (e.g., specialized directives for legal analysis, creative writing, or system architecture).

## 4. The Self-Improving Language
- **LLM-Powered Iteration:** A meta-LLM analyzes the outputs of thousands of MEL prompts to continuously refine the language's vocabulary, suggesting highly effective new slider expressions (e.g., discovering that "using first-principles thinking" is an optimal `Innovate` directive).

## 5. Multi-Agent Orchestration
- **From Prompts to Plans:** Evolving MEL beyond single-prompt generation to become the human-centric language for defining nodes in a reasoning graph. A high-level MEL prompt defines a complex goal, which an agentic system (like OpenClaw) decomposes into atomic tasks, each governed by its own MEL directive.
